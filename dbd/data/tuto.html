Tutorial


This tutorial is a simple demonstration of the 'Smart Random Build Generator' (SRBG) tool


Each tab from SRBG will be briefly described to highlight the main features


Tab 'Configure Perks'

Use this tab to configure perks (default weights are loaded at start)



Weights simply modulate the probability for perks to be selected in generated builds

Higher weights mean that related perks will have higher probability to be selected

Identical weights involve that all perks will have the same probability to be selected

Perks will be disabled if they have a weight of 0


Step1 : select active side ("Survivor" or "Killer")

Step2 : set weight of each perk using various ways:

  • use default weights that are loaded at start
  • click 'reload original weights' button to reload predefined weights for each perk
  • click 'set identical weights' button to set the same weight for each perk
  • click 'open custom perk distribution' button to load custom weights for each perk
  • manual edit of weights directly from table

Step3 : Do not forget to save perk distribution using 'save current perk distribution' button if you manually modified weights from table


Custom weights will be loaded at start if configuration file ('perk_db_custom.txt') is found in working directory


Tab 'Get Random Builds'

Use this tab to set parameters and to generate random builds



Step1 : set number of perks in each build

Step2 : set number of wanted random builds

Step3 : enable/disable random selection of side (bypass previous tab)

Step4 : enable/disable random selection of character

Step5 : enable/disable constraints about care/sprint perks (only available on 'Survivor' side)

Step6 : click 'get random builds' button to generate random builds


Perks and character from 1st random build are graphically displayed in table (it can be automatically saved using the next tab)


Tab 'Database of Builds'

Use this tab to store and manipulate (add/remove) builds in the database

More than 1000 predefined builds are automatically loaded at start



Button 'reload database' : reload original build database

Button 'open database' : open custom build database

Button 'save database' : save current build database

Button 'add random build' : add random build from previous tab to database

Button 'add current build' : add current designed build to database (side/character and 4 distinct perks were previously selected)

Button 'delete builds' : delete selected builds from database


Filter Rows : search/filter database using text-based patterns (case insensitive search, regular expressions can be used)




Button 'random select' : randomly select one saved build from database



You can use this function to randomly select a build within a shortlist (at least 2 selected rows)




Do not forget to save the build database using 'save database' button if you modified it


Custom build database will be loaded at start if file ('build_db_custom.txt') is found in working directory


Finally, you can also suggest here other builds which are not currently present in 'default build database'
